### 【实验目的】

理解和掌握如何基于主动方式和被动方式对目标网站进行CMS信息收集。

### 【知识点】

whatweb、主动信息收集方式、被动信息收集方式

### 【实验原理】

WhatWeb工具支持插件和脚本扩展。支持插件扩展，用于检测特定类型的应用程序，可以在命令行中使用–plugins参数查看所有可用的插件；支持脚本自定义，允许渗透测试人员自行添加识别规则，提高工具的适用性和准确性。WhatWeb工具的使用可以通过在Kali Linux终端中输入whatweb [目标URL]命令进行运行，其中目标URL指代想要进行检测的网络地址。

### 【软件工具】

- 服务器：Windows Server 2008 1台；防火墙 1台；Kali Linux 1台；Centoes7 2台；Windows Server 2016 1台；
- 交换机/路由：交换机 2台；路由器 1台；
- 软件：Whatweb

### 【实验拓扑】

> 注：为展现拓扑完整性，拓扑中为实验的拓扑启动的是子任务实验（中灰色部分表示为未启动状态，蓝色部分表示为启动状态）。

### 【实验预期】

能够分别基于主动和被动方式收集目标系统各网站所使用的CMS版本信息。

### 【实验思路】

1. 基于主动方式使用WhatWeb工具收集门户网站1服务器8080端口和门户网站2服务器80端口站点的CMS版本信息；
2. 基于被动方式收集门户网站1服务器80端口和门户网站3服务器8080端口站点的CMS版本信息。
