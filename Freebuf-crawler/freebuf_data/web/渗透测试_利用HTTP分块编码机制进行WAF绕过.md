## 利用HTTP分块编码机制进行WAF绕过

### 【实验目的】

理解和掌握Burp Suite工具中数据包分块插件的配置和使用；

理解和掌握利用HTTP分块编码机制对目标系统WAF进行绕过的方法。

### 【知识点】

HTTP分块

### 【实验原理】

HTTP分块编码（HTTP Chunked Encoding）是一种HTTP协议的传输编码方式，允许数据以块的形式进行传输，而不需要等待整个消息完全生成。这种编码方式常用于传输动态生成的、大小未知的响应内容，或者用于流式传输数据。使用分块编码时，需要指定HTTP协议标头参数Transfer-Encoding的值。当值为chunked时，表示数据以一系列块的形式进行发送。在这种情况下，Content-Length参数会被忽略，并且在每个块的开头，渗透测试人员需要以十六进制格式添加当前块的长度。HTTP分块编码允许将数据分割成一系列块（chunks），每个块包含了一段数据和该段数据的大小信息。每个块以该块的大小开始，接着是回车换行符（\r\n），然后是数据本身，再接着是另一个回车换行符（\r\n）。最后的块大小为0，表示消息结束。

### 【软件工具】

- 服务器：Windows Server 2008 1台；防火墙 1台；Windows 10 1台；
- 交换机/路由：交换机 2台；路由器 1台；
- 软件：火狐浏览器，BurpSuite

### 【实验拓扑】

### 【实验思路】

1. 访问目标网站并配置代理功能；
2. 抓取请求数据包并进行恶意访问；
3. 在Burp Suite工具中导入数据包分块插件；
4. 对目标网站进行WAF绕过，查询数据表名称信息；
5. 对目标网站进行WAF绕过，查询数据表字段名称信息；
6. 对目标网站进行WAF绕过，查询数据表字段内容信息。

### 【实验预期】

能够通过Brup Suite工具借助数据包分块插件的形式，利用HTTP分块编码机制对目标网站进行WAF绕过，查询出目标系统管理员信息。

### 【实验步骤】

#### 1.访问目标网站并配置代理功能

登录实验拓扑图坐下方的【攻击机2-Windows】，选择并打开虚拟机桌面的火狐浏览器,在浏览器地址栏中输入URL地址：http://202.1.10.34:8080，访问目标网站，如下图所示。

单击火狐浏览器右上角【三】图标，弹出菜单栏，单击选择【选项】，如下图所示。

依次单击【常规】→网络设置的【设置】按钮，进入火狐浏览器代理窗口，如下图所示。

单击【手动配置代理】单选按钮，并单击【确定】按钮，完成浏览器代理服务配置，如下图所示。

双击桌面上的【BurpSuiteCommunity.exe】图标，打开Burp Suite工具，弹出窗口依次单击【Delete】→【Next】→【Start Burp】，如下图所示。

弹出Burp Suite工具窗口后，单击【Proxy】选项卡，并单击【Options】选项卡，查看Burp Suite工具代理配置是否与火狐浏览器代理服务配置一致，如下图所示。

单击【Intercept】选项卡，并单击【Intercept is off】按钮，直至【Interccept is off】按钮变【Interccept is on】按钮，如下图所示。
