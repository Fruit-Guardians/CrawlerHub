针对AI基础设施的网络攻击激增，关键漏洞浮出水面
2025-08-06 01:46:14
所属地 上海
网络犯罪分子正逐渐将目标转向训练、调优和部署现代人工智能（AI）模型的高价值基础设施。过去六个月中，事件响应团队发现了一个暂命名为"ShadowInit"的新型恶意软件家族，专门针对大语言模型（LLM）部署中的GPU集群、模型服务网关和编排管道。

### 新型攻击手段与危害

与早期的加密货币挖矿攻击不同，ShadowInit旨在窃取专有模型权重并暗中篡改推理输出，从而破坏欺诈检测系统和自动驾驶堆栈等下游应用的可靠性。

初步遥测数据显示，ShadowInit通过滥用广泛共享的模型训练笔记本来实施入侵，这些笔记本使用了未固定版本的软件包。当受害者拉取笔记本时，被植入的恶意依赖会下载一个针对NVIDIA CUDA运行时编译的ELF格式投放器。

趋势科技分析师在美国东海岸一个运行A100集群的研究实验室发现异常外发流量后，追踪到该恶意软件与BianLian勒索软件团伙存在关联的威胁组织。该组织涉嫌在暗网论坛以每100MB数据5000美元的价格出售"模型泄露"数据集。

### 直接损失与长期威胁

ShadowInit造成的损失包括直接和长期影响。直接损失包括平均每次入侵消耗6400 GPU小时的算力资源，以及完整性检查导致的强制停机。长期威胁更难量化：被盗的模型权重使攻击者能够制作高度逼真的钓鱼内容，或以极低成本微调竞争模型。

某制造企业案例中，被篡改的视觉模型错误分类了安全关键缺陷，导致生产线停工47分钟，造成约300万美元收入损失。

### 技术分析与防御建议

深入分析显示该恶意软件采用模块化结构。轻量级加载器执行环境检查后，从Jupyter笔记本元数据字段中存储的base64编码块动态重建主载荷。重建的载荷驻留在固定的GPU缓冲区中，可规避传统用户空间扫描器的检测。值得注意的是，加载器会禁用NVIDIA的Compute Sanitizer钩子，阻止对恶意内核的拦截。

攻击者利用AI基础设施通常由DevOps团队而非安全专家监控的特点，植入欺骗性日志条目。例如，ShadowInit伪造kube-audit消息模拟常规自动扩展事件，使真实警报被大多数仪表板忽略。

### 容器侧加载感染机制

ShadowInit首选感染载体是伪装成合法CUDA基础镜像的恶意OCI层。当开发者执行看似无害的docker pull cuda:12.5-base命令时，注册表返回被篡改的清单，实时替换层摘要。

攻击者注册表代理中的Go代码片段展示了摘要替换过程：

```
func rewriteManifest(w http.ResponseWriter, r *http.Request, legit, evil string) {
  body, _ := io.ReadAll(r.Body)
  manifest := bytes.ReplaceAll(body, []byte(legit), []byte(evil))
  w.Header().Set("Content-Type", "application/vnd.oci.image.manifest.v1+json")
  w.Write(manifest) // 发送篡改后的清单
}

```

容器启动后，注入的层以cap_sys_rawio权限挂载/dev/nvidia0，并部署小型eBPF程序拦截GPU DMA写入。这使得攻击者能够在不修改内核代码的情况下获取推理流量的持久只读快照，满足隐蔽性需求。

侧加载层还会在容器命名空间内安排定时任务，定期通过AES-GCM加密通道将被盗张量数据传输到Cloudflare Workers端点，通过伪装成公共模型中心的遥测数据规避出站防火墙规则。

由于文件完整性监控工具通常忽略易变的容器层，且GPU级钩子很少被审计，检测仍具挑战性。防御者可采取以下措施降低风险：强制实施镜像签名验证、固定笔记本中的依赖版本、将GPU固件日志转发到SIEM管道进行异常检测。

趋势科技研究人员建议部署运行时验证代理，能够对实时模型权重进行哈希处理，并每15分钟与已知良好基线进行比较，该策略可在ShadowInit执行后一小时内发现篡改行为。
